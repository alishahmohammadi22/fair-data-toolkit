The FAIR Data Problem in Pharma
70–80% of pharmaceutical research data is never reused. The root cause is a lack of structured, measurable FAIRness.

Without FAIR

  • Data locked in proprietary LIMS with no external identifiers
  • No standard vocabulary — "VIAB" means different things in different systems
  • Access requires knowing who to email, not a documented protocol
  • No licence — legal ambiguity blocks pre-competitive sharing
  • Metadata disappears when the LIMS is decommissioned
  • AI/ML pipelines fail because training data is not machine-readable

With this Toolkit

  • Score all 41 RDA indicators with evidence documentation
  • Identify Essential gaps that block minimum FAIRness
  • Map your organisation to L0–L5 Pistoia maturity levels across 7 dimensions
  • Generate a prioritised remediation roadmap
  • Track progress over time with reusable JSON/CSV exports
  • Lay the groundwork for automated agentic FAIR scoring (Phase 2)
Two Complementary Frameworks
This toolkit implements both the domain-agnostic RDA model and the pharma-specific Pistoia Alliance matrix.

Pistoia Alliance FAIR Maturity Matrix

Pistoia Alliance Life Sciences FAIR Initiative (v1.1, CC BY 4.0)

An organisational maturity model for FAIR implementation in life science organisations. Structured as a 7-dimension × 6-level matrix (L0–L5). Descriptive, not prescriptive. Developed by 20+ experts from AstraZeneca, BMS, Novo Nordisk, ELIXIR, GO FAIR Foundation, and others.

6 maturity levels (L0–L5) 7 organisational dimensions Organisational self-assessment Life science context
Pistoia Maturity Levels (L0–L5)
🏚️
L0 — Life is unFAIR
Junkyard. Data silos, no metadata standards. No FAIR awareness or leadership engagement.
🛍️
L1 — Started the FAIR Journey
Flea market. FAIR awareness starts; first pilots emerge. Identifiers and metadata beginning.
🏪
L2 — Getting FAIR
Street Market. Pilots in place. Data cataloged; local models; POC infrastructure deployed.
🏬
L3 — Pretty FAIR
Specialized Local Markets. Domain-level models; machine interpretation locally; COTS tools deployed.
🛒
L4 — Really FAIR
Hyper Market. FAIR pervasive; GUPRIs; cross-domain standards; enterprise interoperability.
🌐
L5 — FAIRest of Them All
Digital Online Store. Aspirational. Self-describing objects; machine actionability; ecosystem interoperability.

Each level is assessed across 7 dimensions: FAIR data · FAIR leadership · FAIR strategy · FAIR roles · FAIR processes · FAIR knowledge · FAIR tools & infrastructures. Source: pistoiaalliance.github.io/FAIRMaturityMatrix (v1.1, CC BY 4.0)

Install & Quick Start

Install the package

pip install fair-data-toolkit
# With notebook extras
pip install "fair-data-toolkit[notebooks]"

Run a manual assessment

from fair_toolkit import ManualFAIRAssessor, ComplianceScore, EvidenceLevel

assessor = ManualFAIRAssessor(
    dataset_id="DS-001",
    dataset_title="CAR-T Cell Viability Assay",
    assessed_by="Ali Shahmohammadi",
)

assessor.score("RDA-F1-01D", ComplianceScore.NOT_COMPLIANT,
               evidence="Only internal LIMS ID, no global PID")
assessor.score("RDA-F2-01M", ComplianceScore.PARTIALLY_COMPLIANT,
               evidence="Basic metadata present; assay conditions missing")

assessor.print_scorecard()

result = assessor.build_result()
print(f"Overall FAIR score: {result.overall_score}%")

# Get Essential indicator gaps
gaps = result.get_gaps(EvidenceLevel.ESSENTIAL)
for gap in gaps:
    print(f"  ✗ {gap.indicator_id}: {gap.indicator_name}")
Roadmap
Phase 1 delivers the manual assessment toolkit and articles. Phase 2 adds agentic AI scoring.
Phase 1 — Current Release

Manual Assessment Toolkit

  • All 41 RDA FAIR Maturity Model indicators as Pydantic models
  • Pistoia Alliance FAIR Maturity Matrix: 6 levels (L0–L5) × 7 organisational dimensions
  • ManualFAIRAssessor with Rich scorecard output
  • Gap analysis by Essential / Important / Useful priority
  • Five-article Jupyter notebook series
  • CSV and JSON export of assessment results
Phase 2 — In Development

Agentic FAIR Scorer (fair_agent/)

  • MetadataFetcherAgent — parses any REST API (JSON-LD, DCAT, Dublin Core)
  • StructuralAssessorAgent — auto-scores ~20 structural indicators
  • ContentAnalysisAgent (GPT) — scores semantic indicators from text
  • ExternalVerifierAgent — validates ontology terms via BioPortal/OLS
  • GovernanceReporterAgent — generates remediation roadmap
  • Full LangGraph pipeline — input any API, output FAIRAssessmentResult
