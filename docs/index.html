The Problem
FAIR Data in Pharmaceutical R&D

70–80% of pharmaceutical research data is never reused. The root cause is a lack of structured, measurable FAIRness.

Without FAIR

  • Data locked in proprietary LIMS with no external identifiers
  • No standard vocabulary — “VIAB” means different things across systems
  • Access requires knowing who to email, not a documented protocol
  • No licence — legal ambiguity blocks pre-competitive sharing
  • Metadata disappears when the LIMS is decommissioned
  • AI/ML pipelines fail because training data is not machine-readable

With this Toolkit

  • Score all 41 RDA indicators with evidence documentation
  • Identify Essential gaps that block minimum FAIRness
  • Map your organisation to L0–L5 Pistoia levels across 7 dimensions
  • Generate a prioritised remediation roadmap
  • Track progress over time with reusable JSON/CSV exports
  • Lay the groundwork for automated agentic FAIR scoring (Phase 2)
Frameworks
Two Complementary Frameworks

This toolkit implements both the domain-agnostic RDA model and the pharma-specific Pistoia Alliance matrix.

RDA FAIR Maturity Model (2020)

Research Data Alliance · doi:10.15497/rda00050

The authoritative operationalisation of the 15 FAIR sub-principles into 41 measurable indicators. Each has a precise compliance question, priority level, and evidence requirements.

Pistoia Alliance FAIR Maturity Matrix

Pistoia Alliance Life Sciences FAIR Initiative (v1.1, CC BY 4.0)

An organisational maturity model for FAIR implementation in life science organisations. 7-dimension × 6-level matrix (L0–L5). Developed by 20+ experts from AstraZeneca, BMS, Novo Nordisk, ELIXIR, and GO FAIR Foundation.

Quick Start
Install & Get Scoring

Available on PyPI. Requires Python 3.9+.

Terminal
# Install from PyPI
pip install fair-data-toolkit

# With notebook extras
pip install "fair-data-toolkit[notebooks]"

# From source
git clone https://github.com/alishahmohammadi22/fair-data-toolkit
pip install -e ".[notebooks]"
Python
from fair_toolkit import ManualFAIRAssessor, ComplianceScore

assessor = ManualFAIRAssessor(
    dataset_id="DS-001",
    dataset_title="CAR-T Viability Assay",
    assessed_by="Ali Shahmohammadi",
)
assessor.score("RDA-F1-01D", ComplianceScore.NOT_BEING_CONSIDERED,
    evidence="Only internal LIMS ID, no global PID")
assessor.score("RDA-F2-01M", ComplianceScore.UNDER_CONSIDERATION,
    evidence="Basic metadata; assay conditions missing")
assessor.score("RDA-A1.2-01D", ComplianceScore.FULLY_IMPLEMENTED,
    evidence="Azure AD OAuth 2.0 controls access")

result = assessor.build_result()
print(f"Overall FAIR score: {result.overall_score}%")
Web Application
No-Code Assessment Interface

A full-stack web app wrapping fair_toolkit — for data stewards and scientists who prefer a UI over Python.

F / A / I / R Tabbed Interface
Each FAIR principle on its own tab with live completion percentage. Compact rows expand on click for evidence and notes.
Auto-Save — No Lost Work
Changes debounced and persisted to PostgreSQL every 1.2 seconds. Resume any assessment across sessions and users.
Score Gauges & Gap Analysis
Animated SVG gauges for Overall, F, A, I, R. Results page lists all unmet Essential indicators with evidence and action notes.
REST API + OpenAPI Docs
Full FastAPI backend with interactive Swagger docs at /docs. Bulk score upsert and live scoring on every GET.
One-Command Deploy
Single podman-compose up --build starts PostgreSQL, FastAPI, and Vue 3 SPA. Compatible with Docker Compose.
Single Source of Truth
The API imports fair_toolkit directly — all 41 indicators and scoring logic live in the Python package. Zero duplication.

Roadmap
What’s Built. What’s Next.

Phase 1 delivers the manual assessment toolkit. Phase 2 adds agentic AI scoring.

Phase 1 — Released (v0.2.0)

Manual Assessment Toolkit

  • All 41 RDA FAIR Maturity indicators as Pydantic models
  • Pistoia Alliance FAIR Maturity Matrix: 6 levels × 7 dimensions
  • ManualFAIRAssessor with priority-weighted scoring
  • Gap analysis by Essential / Important / Useful priority
  • Five-article Jupyter notebook series
  • CSV and JSON export of assessment results
  • Vue 3 + FastAPI + PostgreSQL web application
  • Published to Zenodo (DOI: 10.5281/zenodo.20274083)
Phase 2 — In Development

Agentic FAIR Scorer (fair_agent/)

  • MetadataFetcherAgent — parses any REST API (JSON-LD, DCAT, Dublin Core)
  • StructuralAssessorAgent — auto-scores ~20 structural indicators
  • ContentAnalysisAgent (GPT) — scores semantic indicators from text
  • ExternalVerifierAgent — validates ontology terms via BioPortal/OLS
  • GovernanceReporterAgent — generates remediation roadmap
  • Full LangGraph pipeline — input any API, output FAIRAssessmentResult
